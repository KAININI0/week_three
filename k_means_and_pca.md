# K-Means聚类模型
## 公式推导
***1. 基本符号定义***
 
- 样本集：X = {x_1, x_2, ..., x_n}
- 聚类中心：u1, u2, ..., uk
- 每个样本 xi 所属簇：ci（取值 1~k）
- 距离：常用欧氏距离
 
***2. 目标函数***
 
K-Means 要最小化簇内平方和（SSE）：
 

J = sum{i=1}^n sum_{j=1}^k I(ci = j) ||xi - uj||^2

 
- {I}() 是指示函数，成立为 1，否则 0
- 含义：所有样本到自己簇中心的距离平方和最小
 
 
***3. 算法迭代推导（两步交替优化）***
 
（1）E 步：固定簇中心，分配样本
 
对每个样本 xi，分配到最近的簇：
 

ci = argmin{j 属于{1,...,k\}} ||xi - uj||^2

 
这一步让 J 关于 ci 最小。
 
（2）M 步：固定簇分配，更新中心
 
对每个簇 j，中心等于该簇所有样本的均值：
 

uj = 1/Nj sum{i: ci=j} xi

 
- Nj：第 j 簇样本数量
这一步让 J 关于 uj 最小。
 
 ***4. 为什么中心是均值？（简单推导）***
 
对簇 j，对 uj 求导并令导数为 0：
 

 ∂/ ∂uj sumi:ci=ji (xi-ui)=0

 
展开：
 ui=1/Nj sumxi
 
所以簇中心必须是均值。
 
***5. 完整算法流程（公式版）***
 
1. 随机初始化 k 个中心 u1,...,uk
2. E 步：对所有 i，计算 ci = argminj\|xi-uj|^2
3. M 步：对所有 j，更新 uj = 1/Njsum_{i:ci=j}xi
4. 重复 2~3，直到中心不再变化或达到迭代次数
 
### 总结
 
K-Means = 交替最小化簇内平方和
 
- 固定中心 → 分配样本
- 固定分配 → 更新均值中心
直到收敛。

## 优劣势
### 优点
 
1. 简单、速度快、效率高
时间复杂度接近线性 O(tkn)，适合大数据集。
2. 可解释性强
原理清晰，结果直观，簇中心就是均值。
3. 实现方便、应用广
几乎所有机器学习库都自带，图像、用户分群、特征压缩常用。
4. 对凸簇、密集球形数据效果好
数据满足高斯分布、簇大小相近时表现稳定。
 ### 缺点
 
1. K 值需要人工预先指定
不知道聚几类就很难用，需要手肘法、轮廓系数辅助。
2. 对初始聚类中心敏感
初始化不好容易陷入局部最优，结果不稳定。
3. 对异常值、噪声非常敏感
outliers 会严重拉偏簇中心。
4. 只能发现球形/凸形簇
非凸、环形、细长形簇、密度不均的数据效果很差。
5. 默认簇大小、密度相近
一簇特别大、一簇特别小的时候容易分错。
6. 对量纲敏感
特征数值范围差异大时必须标准化，否则距离失真。

# PCA降维技术
***1. 缓解维度灾难***
- 通过降维，将高维特征映射到低维空间，在保留主要信息的前提下大幅减少特征数量。
- 降低模型复杂度，减少过拟合风险，提升训练速度，避免高维下距离失效、计算量爆炸等维度灾难问题。
- 消除特征冗余，使数据更紧凑、密度更高，便于后续聚类、分类等算法有效工作。
***2. 数据预处理中的作用***
- 去相关性：消除特征之间的多重共线性，使新特征彼此线性无关。
- 降噪：保留主成分、舍弃噪声占比较大的次要成分，实现数据平滑与提纯。
- 标准化与归一化辅助：PCA 对量纲敏感，常配合标准化使用，间接推动数据规范化。
- 可视化：将高维数据降至 2D/3D，便于观察数据分布、聚类结构与异常点。
- 特征提取：自动提取综合特征，替代人工设计，提升后续模型效果与稳定性。
